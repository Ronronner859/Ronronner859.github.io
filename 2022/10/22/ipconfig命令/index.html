ipconfig命令

|阅读量:

ipconfig命令

  1. ipconfig/release ：释放 IP 地址租约。

  2. ipconfig/ flushdns ：清除本地 DNS 缓存。

  3. ipconfig/ displaydns ：显示本地 DNS 内容。

  4. ipconfig/ registerdns ： DNS 客户端手工向服务器进行注册。

    其他：

  5. ipconfig /all ：显示本机 TCP/IP 配置的详细信息。

  6. ipconfig /renew ： DHCP 客户端手工向服务器刷新请求。

  7. ipconfig /showclassid ：显示网络适配器的 DHCP 类别信息。

  8. ipconfig /setclassid ：设置网络适配器的 DHCP 类别。

  9. ipconfig /renew “LocalAreaConnection”：更新“本地连接”适配器的由 DHCP 分配 IP地址的配置。

  10. ipconfig /showclassidLocal*：显示名称以 Local 开头的所有适配器的 DHCP 类别 ID 。

  11. ipconfig /setclassid “LocalAreaConnection”TEST ：将“本地连接”适配器的 DHCP类别 ID 设置为 TEST 。

  12. ipconfig ( linux: ifconfig) (显示 TCP/IP 网络配置值，如： IP 地址， MAC 地址，网关 地址等)。

  13. tracert(linux: traceroute)：用于确定 IP 数据包访问目标所采取的路径，若网络不通， 能定位到具体哪个结点不通。

  14. netstat ：用于显示网络连接、路由表和网络接口信息。

  15. nslookup(查询 DNS 记录)。

端口地址

POP3 协议采用 C/S 模式进行通信， POP3 需要 TCP 连接的支持，当客户机需要服务时，客户端软件与 POP3 服务器建立 TCP 连接。
POP3 使用的端口号为 110,用以接收邮件报文。该报文采用的传输层协议是 TCP 。
常用协议端口号情况如下：

  • POP3 ： 110 端口，邮件收取。
    SMTP ： 25 端口，邮件发送。
    FTP ： 20 数据端口/21 控制端口，文件传输协议。
    HTTP ： 80 端口，超文本传输协议，网页传输。
    DHCP ： 67 端口， IP 地址自动分配。
    SNMP ： 161 端口，简单网络管理协议。
  • DNS ： 53 端口，域名协议，记录域名与 IP 的映射关系。
    TCP ：可靠的传输层协议。
    UDP ：不可靠的传输层协议。
    ICMP ：因特网控制协议， PING 命令来自该协议。
    IGMP ：组播协议。
    ARP ：地址协议， IP 地址转换为 MAC 地址。
    RARP ：反向地址协议， MAC 地址转 IP 地址。
